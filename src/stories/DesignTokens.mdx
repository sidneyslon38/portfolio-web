# Design Tokens & SCSS Utilities

This page documents every CSS custom property declared in `:root` inside
`src/app.scss`, as well as the SCSS variables and mixins available in
`src/lib/styles/`. Use these tokens in your components instead of hard-coded
values to keep the design system consistent.

## Typography

Defined in `src/app.scss` — import with `var(--token-name)`.

### Font Families

| Token          | Value                                                                                                 | Usage                |
| -------------- | ----------------------------------------------------------------------------------------------------- | -------------------- |
| `--font-serif` | `'PT Serif', Georgia, 'Times New Roman', Times, serif`                                                | Headings (`h1`–`h6`) |
| `--font-sans`  | `'Droid Sans', system-ui, sans-serif, ...` _(abbreviated; see `src/app.scss` for full fallback list)_ | Body text, UI labels |

### Root Font Sizes

| Token                     | Value  | Notes                                                       |
| ------------------------- | ------ | ----------------------------------------------------------- |
| `--font-size-root`        | `16px` | Applied to `html` on desktop                                |
| `--font-size-root-mobile` | `15px` | Applied to `html` on screens ≤ 768 px via `@include mobile` |

### Font Weights

| Token                     | Value | Usage                           |
| ------------------------- | ----- | ------------------------------- |
| `--font-weight-light`     | `300` | Decorative light text           |
| `--font-weight-normal`    | `400` | Default body and heading weight |
| `--font-weight-medium`    | `500` | Slightly emphasized UI text     |
| `--font-weight-semibold`  | `600` | Labels, nav items               |
| `--font-weight-bold`      | `700` | Bold body text, bold headings   |
| `--font-weight-extrabold` | `800` | Display callouts, big numbers   |

### Line Heights

| Token               | Value  | Usage                       |
| ------------------- | ------ | --------------------------- |
| `--leading-tight`   | `1.15` | Compact multi-line headings |
| `--leading-snug`    | `1.3`  | Sub-headings, titles        |
| `--leading-caption` | `1.4`  | Captions, image credits     |
| `--leading-normal`  | `1.6`  | Default body text           |
| `--leading-relaxed` | `1.75` | Long-form article text      |
| `--leading-loose`   | `1.8`  | Highly readable prose       |
| `--leading-heading` | `1.2`  | `h1`–`h6` default           |

### Letter Spacing

| Token                    | Value     | Usage                       |
| ------------------------ | --------- | --------------------------- |
| `--letter-spacing-tight` | `-0.01em` | Display headlines           |
| `--letter-spacing-wide`  | `0.02em`  | All-caps labels             |
| `--letter-spacing-wider` | `0.05em`  | Small-caps or overline text |

## Dropcap

Defined in `src/app.scss` — import with `var(--token-name)`.

Apply `class="dropcap"` to a `<p>` inside `<ArticleBody>` to render a large
initial letter. At runtime, the first character is wrapped in a
`<span class="dropcap-letter">` and styled with an animated blue gradient
matching the site header.

| Token                    | Value                 | Usage                                         |
| ------------------------ | --------------------- | --------------------------------------------- |
| `--dropcap-font-size`    | `3.5em`               | Size of the initial letter                    |
| `--dropcap-line-height`  | `1`                   | Line-height so the letter aligns with 2 lines |
| `--dropcap-margin-right` | `0.05em`              | Space between letter and text                 |
| `--dropcap-color`        | `var(--color-accent)` | Fallback color of the initial letter          |

The dropcap letter uses the global `dropcap-gradient` keyframe animation
(defined in `src/app.scss`), which sweeps through
`--color-cuny-blue-dark` → `--color-accent` → `--color-cuny-blue-light`
over 6 seconds, identical to the `gradient-sweep` animation on the site header.

## SCSS Variables

Defined in `src/lib/styles/_variables.scss` — available after `@use '$lib/styles' as *`.

CSS custom properties cannot be used inside `@media` conditions, so breakpoints
are stored as plain Sass variables.

| Variable         | Value    | Usage                   |
| ---------------- | -------- | ----------------------- |
| `$breakpoint-sm` | `768px`  | Mobile/tablet threshold |
| `$breakpoint-lg` | `1200px` | Desktop threshold       |

## SCSS Mixins

Defined in `src/lib/styles/_mixins.scss` — available after `@use '$lib/styles' as *`.

Use these mixins instead of raw `@media` queries so breakpoint values stay in
one place and are easy to update.

### `@include mobile { … }`

Wraps styles in `@media (max-width: 768px)`. Use for overrides on small screens.

```scss
@use '$lib/styles' as *;

.headline {
  font-size: var(--font-size-6xl);

  @include mobile {
    font-size: var(--font-size-4xl); // smaller on phones
  }
}
```

### `@include tablet { … }`

Wraps styles in `@media (min-width: 768px)`. Use for tablet-and-up overrides.

```scss
@use '$lib/styles' as *;

.sidebar {
  display: none;

  @include tablet {
    display: block; // show sidebar on tablet and up
  }
}
```

### `@include desktop { … }`

Wraps styles in `@media (min-width: 1200px)`. Use for large-screen enhancements.

```scss
@use '$lib/styles' as *;

.chart-grid {
  grid-template-columns: 1fr;

  @include desktop {
    grid-template-columns: repeat(3, 1fr); // 3-column layout on desktop
  }
}
```

### Importing the Style Utilities

All variables and mixins are exported from the barrel file
`src/lib/styles/_index.scss`. Import everything in a single line:

```scss
@use '$lib/styles' as *;
```
